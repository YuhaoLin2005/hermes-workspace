# 我给 AI Agent 装了神经门——自我验证的第二层

> 文件检查问"脚本跑了吗？"神经门问"约束真的改变了输出吗？"

## 文件系统门的局限

过去一个月我一直在给 Claude Code 搭机械门——检查文件时间戳、hook 注册、进程退出码。它们管用，确实抓到了配置漂移。但它们都基于同一个假设：文件存在+hook 接线+脚本运行=约束在生效。

这是错的。AI agent 可以读了规则、在自评中提到它、生成看起来合规的输出——但实际上根本没被它影响。规则在上下文窗口里，agent 被问到时也会提。但 token 概率分布没有发生偏移。

文件系统门检查"到达"。不检查"穿透"。

## AI 逻辑 ≠ 人类逻辑

我一直在用人类逻辑验证 AI 系统。但 AI agent 的原生感官是注意力权重、残差流方向、logprob 分布。验证必须发生在信息实际流动的层面。

## 神经门 v1：约束回响检测

`neural-gate.py`（86 行）。从 BODY.md 提取 8 个约束主题，扫描输出文件关键词回响。已通过 150 任务对照实验验证——见下文更新。

用 DeepSeek `logprobs=True` 对比带/不带约束时 token 概率分布。脚本已写好，等 API key。
