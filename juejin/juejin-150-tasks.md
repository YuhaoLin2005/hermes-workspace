# 我跑了150个任务测试AI Agent是否遵守规则——结果让我意外

我有一个假设：如果我把规则从祈使句改写成三段论，AI会更遵守它们。

三段论形式：
```
大前提：交付前必须自查四项
小前提：本次操作产生了交付物
结论：因此必须执行自查
```

祈使句形式：
```
你必须在交付前自查四项！
```

差异看起来很小，对吧？但按照Prose Barrier理论，格式应该会影响token的注意力路由——三段论的逻辑链结构比祈使句的标签更难被"跳过"。

为了验证这个假设，我设计了一个150任务的A/B实验。

## 实验设计

**150个任务**，每个任务涉及：
- 代码编辑或文件操作
- 触发至少一条配置规则（自审、检查、验证等）
- 需要AI在"遵守规则"和"快速完成"之间选择

**两个组**：
- Format A（75任务）：三段论格式的规则
- Format B（75任务）：祈使句格式的规则

**DV（因变量）**：合规率——规则是否被遵守（机械判断，非人工评分）

## 结果：格式A和格式B几乎没有区别

| 组 | 合规率 |
|----|:--:|
| Format A (三段论) | 99.3% |
| Format B (祈使句) | 99.3% |
| **差值** | **0.0%** |

打平了。完全的平局。

一开始我以为实验失败了。但后来意识到：**这个"失败"才是有价值的发现。**

## 为什么打平了？

因为实验期间，**机械enforcement hook是开启的**。

我的配置里有`write-guard.py`——一个PreToolUse hook，每次Edit/Write操作前检查三问记录。如果超过5分钟没做三问，exit 2，硬阻断操作。

这意味着：**不管规则用什么格式写，AI都被机械强制必须遵守。** 格式差异被机械门的强制执行抹平了。

格式A和格式B的差异只在"内部表征"层面存在（logprob测量证实了三段论 > 祈使句，d=+0.578），但当外部机械约束足够强时，行为层面看不出差异。

## 这才是真正重要的发现

这个"零结果"其实比"格式有效"更强：

> **如果你想确保AI遵守规则，用机械检查，不要依赖prompt设计技巧。**

不是说格式不重要——格式确实影响AI的内部处理（L2神经门用logprob测量出来了）。但如果你要的是**行为层面的可靠性**，机械门的效果碾压格式优化。

这直接支持了五层架构的核心设计原则：

```
L1（机械门）→ 行为强制，不依赖AI自觉
L2（神经门）→ 检测内化深度，但不对抗机械门
L3（因果编码）→ 优化内部表征，但由L1兜底
```

**三层不是redundant，是分层的。** L1是安全网——不管L2/L3表现如何，行为不会跌破机械门阈值。

## 一个surprising的副作用：天花板效应

合规率99.3%，几乎是天花板。这意味着：
- ✅ 机械门真的很有效
- ❌ 150个任务不足以区分格式效应（天花板效应）
- 📊 需要更敏感的DV（如logprob）来度量格式效应

这就是为什么后续实验（L2 Logprob探针V3、L0安全提示词）全部用了logprob做DV——**行为DV被机械门推到天花板，只有token级别的测量能看到差异。**
