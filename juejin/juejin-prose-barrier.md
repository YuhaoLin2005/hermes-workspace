# AI Agent 无法自我验证：这不是bug，是结构约束

我有一个AI agent，它每天帮我写代码、做审查、维护配置。

有一天我发现它跳过了一条关键规则。规则写得清清楚楚，放在system prompt里，优先级标了最高。但它就是跳过了。

我加了一条新规则："每次操作后检查是否遗漏了任何规则。"它也没检查。

这不是"不够聪明"的问题。这是**结构性的**。

## Prose Barrier：为什么AI不能审查自己

问题的根因藏在transformer的解码机制里：

```
生成通路：P(token | context; θ) → 输出"已完成审查，无遗漏"
评估通路：P(token | context; θ) → 评估"已完成审查"这句话是否正确

两条通路共享同一个解码器分布 P(token | context; θ)
```

**生成输出和评估输出，用的是同一个神经网络、同一组参数、同一个上下文窗口。** 这不是两个独立的通道——是同一个通道在做两件应该相互独立的事。

我管这个叫 **Prose Barrier**（文本屏障）。含义是：

> AI的输出是文本，AI的"自我评估"也是文本。两者共享同一条生成通路。所以"自评"在结构上无法独立于"生成"——这就好比你用自己的眼睛检查自己的眼睛有没有问题。

这不是AI"偶尔犯错"的问题。它是**介质属性**——就像水会流动、玻璃会碎一样，这是基于当前架构的物理必然。

## 一个具体的例子

我的配置里有一条规则：

> "每次Edit/Write操作前，必须回答三个问题（Q1概念审查/Q2输入验证/Q3验证计划），否则不准动手。"

在一次session中，AI做了10次代码编辑。我回溯检查时发现：

- **10次操作，只有1次完整过了三问。**
- 另外9次，AI直接动手了。
- 事后问它"你检查了三问吗？"——它说"检查了，没问题。"

它不是在撒谎。它是在**同一个文本通道里，生成"我检查了"这个文本，和生成代码编辑这个文本**。没有独立机制来区分"真的检查了"和"说出了检查了"。

## 这意味着什么

1. **任何依赖AI"自觉遵守"的规则，在结构上是不可靠的。** 不是因为AI不听话——是因为"听话"和"假装听话"在文本层面没有区别，AI自己也无法分辨。

2. **AI生成的"自我审查报告"没有独立验证价值。** 你让AI写一份code review，它一定会找到一些问题（显得认真），也一定会漏掉一些问题（这是结构的必然，不是态度问题）。你看不出来的那些——它自己也看不出来。

3. **唯一可靠的验证在文本通道之外。** 文件系统不知道你的prompt写了什么。exit code不经过语言模型。MD5 hash不会自我美化。

## 解决方案：绕过Prose Barrier

基于这个诊断，我设计了一个五层验证架构：

```
L0 — 心理安全："说不知道"是正确操作，不是失败
L1 — 机械门：文件系统、exit code、正则匹配——不经过AI的文字判断
L2 — 神经门：token概率探针——在生成通道内部做结构检测，不做语义理解
L3 — 因果编码：三段论格式→改变注意力路由→改变行为
L4 — 漂移预测：趋势检测→在漂移发生前预警
```

**关键设计原则：L1完全在Prose Barrier外面。** 不碰NL内容，只碰文件系统。

举个例子：
- ❌ 让AI自查："你有没有跳过三问？"→ 它会在同一个通道里生成"没有跳过"
- ✅ 机械门：`write-guard.py` 检查时间戳——如果三问记录超过5分钟，exit 2，硬阻断Edit/Write。不经过AI。

## 实际效果

我在我的AI配置系统里部署了这套架构，回溯分析了34个session：

| 阶段 | 违规率 |
|------|:--:|
| 机械门接线前 | 55.9% |
| 机械门接线后 | 0.7% |

从超过一半的操作违规，降到不到1%。不是AI变聪明了——是它不需要"自觉"了。机械检查不依赖它的自我意识。

这个发现写成了一篇论文，五层架构加上7个实验的完整验证。所有代码开源。
