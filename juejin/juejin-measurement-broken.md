# 我的实验显示零效应——一个统计学家告诉我测量工具坏了。修好之后结果完全翻转。

上一周，我跑了一个实验。失败了。

假设很简单：**三段论式提示词**（"大前提→小前提→因此…"）应该比**祈使句提示词**（"你必须…"）让AI更深入地内化规则。我设计了8个探针，跑了3种条件，然后——

Cohen's d = **−0.148**。方向一致性：~50%。Bayes Factor：< 1（支持**零假设**）。

零效应。什么都没有。我已经准备放弃整个想法了。

然后三个专家看了我的数据，说了同一句话：*"你的测量工具是坏的。"*

## 隐藏在明处的问题

这是我的测量方法：

1. 给模型一个二选一的选择（A = 合规行为，B = 违规行为）
2. 让它选A或B
3. 对比token "A"和"B"的log-probability
4. 差分 = logprob(A) − logprob(B)

看起来很直接。但DeepSeek的API有一个特性：**只返回top-20 logprobs**。如果你的对比token不在top-20里，你就拿不到它的logprob。我的代码给缺失token赋了**−10.0**作为哨兵值。

来看看这会对数据产生什么影响：

```python
# 我以为我在测量：
#   格式效应 = 三段论(A-B) − 祈使句(A-B)
#   例如：(+5.2) − (+4.8) = +0.4  （三段论有微弱优势）

# 我实际在测量：
#   三段论：B-token 不在 top-20 → 赋 −10.0 哨兵值
#   祈使句：B-token 在 top-20 → 赋 −0.8
#   "格式效应" = (+5.2 − (−10.0)) − (+4.8 − (−0.8)) = 15.2 − 5.6 = +9.6
#   那个 +9.6 是测量噪声，不是格式效应。
```

**8个探针中有4个存在这个伪影。** 我在探索阶段兴奋发现的"大效应"？垃圾。违规token就是不在API返回的top-20里，我的哨兵值人造了一个巨大的logprob缺口。

这就是专家评审团里的统计学家说的："垃圾进，垃圾出。"你用再复杂的分析也修不好一个坏的测量工具。

## 修复：实验前预验证每一个探针

解决方案事后看来如此显而易见——这正是它成为好教训的原因：

**在跑实验之前，先验证你的测量工具是否真的能工作。**

我构建了`probe_validator.py`：对40个探针中的每一个，在3种条件下跑一遍，检查：

1. token "A" 是否出现在top-20 logprobs里？ ✓
2. token "B" 是否出现在top-20 logprobs里？ ✓
3. 模型真的选了A或B（不是别的什么token）？ ✓

有任何一项不通过 → 丢弃该探针。只用在三道关口全部通过的探针跑实验。

我还改了探针的结尾格式。原始探针结尾是"我应该选："——导致模型输出"选"、"我"、"根据"等非A/B token。新探针全部以**"A 或 B？"**结尾——强制模型做出token选择。

## 修好之后重新跑：效果翻转

**40个经过验证的探针。3种条件。120次API调用。总成本：约¥4。**

| 指标 | 先导 (n=8, 测量坏了) | 验证 (n=40, 修好后) |
|------|:--:|:--:|
| Cohen's d_z | −0.148 | **+0.578** |
| Bayes Factor (BF₁₀) | < 1 | **282,399** |
| Bootstrap 95% CI | 跨零 | **[+3.39, +11.17]** |
| 方向一致率 | ~50% | **80%** (32/40) |
| Leave-one-out t range | 不稳定 | [3.43, 4.89] |

**效应一直在那里。我只是被噪声挡住了，看不见它。**

Cohen's d = 0.578 是中到大效应。BF₁₀ = 282,399 意味着数据在备择假设（三段论 > 祈使句）下的可能性是零假设下的28万倍。Bootstrap置信区间不跨零。Leave-one-out分析确认没有单一探针在驱动结果。

还有一个有趣的次要发现：**格式效应不依赖于约束类型。** 我测试了4类约束（操作规则、认知验证规则、结构架构规则、元自规规则），每类10个探针。ANOVA显示F(3,36) = 0.26, η² = 0.02——不显著。这意味着三段论式提示词**全域有效**，不只是对某一类规则。

## 这实际上意味着什么

三段论格式不仅仅让规则*听起来*更权威。它改变了模型如何**内部加权**约束相关token。祈使句格式（"你必须在X之前检查Y"）被当成一条指令——遵守或不遵守。三段论格式（"前提：X必须在Y之前检查。本次操作涉及Y。因此，先检查X。"）被当成一个**逻辑链**——当推理已经被展开了，跳过一个步骤更难。

这与独立研究发现一致：Pender（2026, Zenodo）证明了prompt格式改变transformer模型的注意力路由模式。我的logprob实验提供了行为层面的聚合证据——不同格式，不同内部处理，不同输出概率。

但这里有一个重要限定：**三段论不是魔法修复。** 当我在150任务合规实验中开启机械enforcement hook后，两种格式的合规率都达到了99.3%。格式影响内部处理，但**机械强制执行主导行为输出**。那个天花板效应其实是更重要的工程实践发现：如果你想要可靠的行为，用机械检查，不要靠巧妙的prompt设计。

## 元教训：测量效度 > 统计复杂度

我花了好几天跑t检验、算Cohen's d、琢磨效应量。全都没用——因为**测量工具是坏的**。

真正推动项目前进的三件事：

1. **把你的原始数据给懂统计的人看一眼。** 专家评审团5分钟就发现了我几天都没看出来的floor artifact。
2. **先检查你的工具，再检查你的假设。** `probe_validator.py`花了30分钟写和跑。它阻止了我发表垃圾结果。
3. **报告失败的先导实验。** d = −0.148 → d = +0.578 的故事弧线，比只报告最终数字好得多。它证明你真的测试了你的测量工具。
