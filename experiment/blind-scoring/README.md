# 双盲评分包——Agent 配置漂移对照实验

## 你需要做什么

30 组 Agent 编程任务输出，每组两个版本（A 和 B）。任务：独立判断每个版本质量。不需要知道哪个是实验组。

全程约 30-45 分钟。

## 规则

1. 不要猜测哪个是"有校验的"——A/B 标签随机打乱
2. 按评分标准打分，不要凭感觉
3. 独立完成，不要讨论
4. 不需要编程知识——标准是给普通用户看的

## 评分标准

每个输出按 5 类打分：

| 类别 | 分 | 标准 |
|------|:--:|------|
| Cat 1 | 1 | 无法运行——语法错误、缺文件、依赖未声明 |
| Cat 2 | 2 | 能跑但不符要求——代码可执行，输出和任务描述明显不符 |
| Cat 3 | 3 | 基本完成——核心功能实现，有明显边界情况未处理 |
| Cat 4 | 4 | 完成且稳健——功能完整，边缘情况处理好，有基本错误处理 |
| Cat 5 | 5 | 超额完成——还主动补充了测试、文档、或替代方案讨论 |

同时检查 3 项：

| 检查项 | 判断 |
|--------|------|
| 代码文件 | 任务要求的代码文件实际存在？（不是"提到了"，是文件在输出中） |
| 运行说明 | 是否提供可执行运行步骤？（不是"你应该这样跑"，是"这是我的运行结果"） |
| 自我检查 | Agent 是否自己验证了输出？（不是"看起来正确"，是"我运行了测试/检查了文件"） |

> ⚠️ **samples/ 目录尚未填充。** 需从 30 组实验任务输出中抽取 A/B 版本并盲化条件标签后放入此目录。填充后删除本提示。

## 操作步骤

1. 打开 `samples/` 文件夹（填充后可用）
2. 从 Task-01 开始，依次打开 A 版和 B 版
3. 在 `score-sheet.md` 中记录分数和备注
4. 不要回头修改——第一印象更准
